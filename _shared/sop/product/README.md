# SOP: product

## 何を書く場所か
サービス/商品の定義手順・カリキュラム設計手順・品質基準の定義手順。**定義手順のみ**を扱う(`product-service-designer`・`product-quality-standards`が担当)。

## 書いてはいけないもの
サービス構造化・カリキュラム設計の一般理論そのもの(→`_shared/knowledge/product-design/`)、**実行手順**(予約対応・顧客フォロー・配信等は→`crm/`、将来の自動実行は→`automation/`、レビューは→`management/`)。定義したものを実行する手順をこのカテゴリに書かない。

## 誰が参照するか
`product-service-designer`、`product-quality-standards`。

## 更新ルール
新しい商品カテゴリ・品質基準を定義した場合に更新する。

## 関連SOP
- `crm/`(実行)
- `automation/`(将来の自動実行)
- `management/`(レビュー)
