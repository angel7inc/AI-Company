# Knowledge: product-design

## 1. 何を保存する場所か
サービス構造化理論(役務提供をどう商品化するかの一般理論)・カリキュラム設計理論(教育コンテンツの構成・学習設計の一般理論)。両者は「顧客に価値を届ける構造をどう設計するか」という同じ対象を扱うため、1カテゴリに統合している。

## 2. 保存してはいけない情報
特定ブランドの実際の商品ラインナップ・価格(→`product-service-designer`のブランド別記録、価格は`_company/brands/{ブランドID}/brand-brief.md`)。

## 3. 誰が参照するか
`product-service-designer`、`product-quality-standards`。

## 4. 更新ルール
有効なサービス構造化・カリキュラム設計のパターンを追加・更新する。

## 5. 推奨する情報源(Sources)
`book`, `industry-report` ― 具体例: サービスデザインの専門書、インストラクショナルデザインの理論

## 6. 関連Knowledge
- `marketing/`
- `business-strategy/`

## 7. 機密情報の扱い
一般理論のため`public`/`internal`。

## 8. 重複防止ルール
同じ設計パターンの重複記事を作らず、既存記事へ追記する。
