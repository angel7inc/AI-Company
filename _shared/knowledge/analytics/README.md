# Knowledge: analytics

## 1. 何を保存する場所か
分析手法・指標の定義・統計の基礎知識(保存率・シェア率等の指標がなぜ重要かという一般理論)。

## 2. 保存してはいけない情報
実際のKPI実績データ・売上等の数値そのもの(→`businesses/{事業}/data/`、`reports/`)。

## 3. 誰が参照するか
`ig-analytics`、将来の全事業の分析担当エージェント。

## 4. 更新ルール
新しい分析手法・指標の考え方を学んだら追加する。

## 5. 推奨する情報源(Sources)
`book`, `industry-report`, `research-paper` ― 具体例: マーケティング分析の専門書、各プラットフォームの公式インサイトドキュメント

## 6. 関連Knowledge
- `marketing/`
- `instagram/`
- `youtube/`

## 7. 機密情報の扱い
分析手法そのものは`public`/`internal`。実績数値と紐づく分析事例は事業側のレポート、または将来の`case-studies`(タイプ別フォルダ)に置く。

## 8. 重複防止ルール
指標の定義は1指標につき1箇所にまとめ、複数記事に同じ定義を書き散らさない。
