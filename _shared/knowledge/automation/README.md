# Knowledge: automation

## 1. 何を保存する場所か
自動化の設計パターン・ワークフロー理論(段階的自動化の考え方、MCP連携の一般パターン、API接続設計パターン、Scheduler(定期/イベント/条件実行)設計パターン、実行監視・ログ設計パターンを含む。Automation Divisionが扱うすべての設計理論をこの1カテゴリに集約し、細分化しない)。

## 2. 保存してはいけない情報
実際のAPIキー・認証情報・接続設定の実値(→`.env`、`mcp/configs/`)、特定事業の自動化実装コードそのもの。

## 3. 誰が参照するか
COO、将来の自動化担当エージェント全般。

## 4. 更新ルール
新しい自動化パターンを試して有効性が確認できたら追加・更新する。

## 5. 推奨する情報源(Sources)
`official`, `documentation`, `experiment` ― 具体例: MCP公式ドキュメント、各種ワークフロー自動化ツールの公式資料

## 6. 関連Knowledge
- `ai/`
- `claude-code/`
- `analytics/`

## 7. 機密情報の扱い
認証情報・接続設定の実値は絶対に記載しない。設計思想・パターンのみを記載する。

## 8. 重複防止ルール
既存の自動化パターン記事と重複しないか確認し、類似パターンは既存記事に追記して統合する。
