# Knowledge: branding

## 1. 何を保存する場所か
ブランディングの一般理論・フレームワーク(ブランドピラミッド、世界観設計の考え方等)。個別ブランドの実データ(Tomo_Angel7のコンセプト等)は`_company/brands/`へ置く。

## 2. 保存してはいけない情報
特定ブランドの未公開の実データ(コンセプト、KPIの実値等)。

## 3. 誰が参照するか
`ig-strategy`、新規ブランド設計時のCOO。

## 4. 更新ルール
新しいブランドを立ち上げる際に得た知見は、ブランド固有の情報を除いて一般化しここへ還元する。

## 5. 推奨する情報源(Sources)
`book`, `industry-report` ― 具体例: ブランディングの定番書籍、業界の成功事例分析

## 6. 関連Knowledge
- `marketing/`
- `design/`
- `copywriting/`

## 7. 機密情報の扱い
一般理論は`public`/`internal`。個別ブランドの未公開情報は絶対に含めない。

## 8. 重複防止ルール
同じフレームワークの重複記事を作らず、既存記事に統合する。
