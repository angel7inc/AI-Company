# Knowledge: crm

## 1. 何を保存する場所か
CRM運用理論(セグメンテーション・ステップ配信設計の考え方)とLTV/リテンション理論(リピート・クロスセル・アップセル・紹介の一般理論)。両者は「顧客との継続関係をどう伸ばすか」という同じ対象を扱うため、1カテゴリに統合している。

## 2. 保存してはいけない情報
特定ブランドの実際のセグメント・配信内容・顧客データ(→`businesses/{事業}/data/`)、実際のリピート率等の実績数値(→`_company/kpi/`)。

## 3. 誰が参照するか
`revenue-crm-manager`、`revenue-ltv-manager`。

## 4. 更新ルール
効果が確認できたCRM/LTV施策のパターンを追加・更新する。

## 5. 推奨する情報源(Sources)
`book`, `industry-report`, `experiment` ― 具体例: CRM/LTVマーケティングの専門書、業界の成功事例分析

## 6. 関連Knowledge
- `marketing/`
- `psychology/`
- `analytics/`

## 7. 機密情報の扱い
一般理論は`public`/`internal`。実際の顧客データ・配信リストは絶対に含めない。

## 8. 重複防止ルール
同じ施策パターンの重複記事を作らず、既存記事へ追記する。
