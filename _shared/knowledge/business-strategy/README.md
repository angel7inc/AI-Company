# Knowledge: business-strategy

## 1. 何を保存する場所か
成長判断・撤退判断・事業ポートフォリオ理論など、恒久的に使える一般的な経営戦略理論(特定事業の実際の戦略決定そのものではない)。

## 2. 保存してはいけない情報
特定事業・特定ブランドの実際の戦略決定内容(→`management-strategy`のレポート、`_company/reports/`)、KPI定義そのもの(→`_shared/knowledge/analytics/`)。

## 3. 誰が参照するか
`management-strategy`、`core-coo`。

## 4. 更新ルール
実務で有効性が確認できたフレームワークを追加・更新する。

## 5. 推奨する情報源(Sources)
`book`, `industry-report` ― 具体例: 経営戦略の定番書籍、成長/撤退判断の業界分析

## 6. 関連Knowledge
- `marketing/`
- `analytics/`

## 7. 機密情報の扱い
一般理論は`public`/`internal`。自社の実際の戦略判断・未公開の数値は含めない。

## 8. 重複防止ルール
同じフレームワークの重複記事を作らず、既存記事へ追記する。
