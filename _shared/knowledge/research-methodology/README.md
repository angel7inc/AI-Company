# Knowledge: research-methodology

## 1. 何を保存する場所か
情報源評価・引用作法・ファクトチェック手法・一次/二次情報の区別に関する一般理論。市場・消費者・トレンド・競合分析の「対象領域の理論」ではなく、「調査の進め方・情報の扱い方」そのものの理論。

## 2. 保存してはいけない情報
実際の調査結果・引用元リストそのもの(→各エージェントのレポート、`businesses/{事業}/content/research-log.md`等)。

## 3. 誰が参照するか
`intelligence-research-quality-checker`、`intelligence-market`、`intelligence-consumer`、`intelligence-trend`。

## 4. 更新ルール
有効なファクトチェック手法・情報源評価基準を追加・更新する。

## 5. 推奨する情報源(Sources)
`book`, `documentation` ― 具体例: リサーチ手法の専門書、ジャーナリズムのファクトチェック手法

## 6. 関連Knowledge
- `business-strategy/`
- `analytics/`
- `ai/`

## 7. 機密情報の扱い
一般理論のため`public`/`internal`。

## 8. 重複防止ルール
同じ評価手法の重複記事を作らず、既存記事へ追記する。
