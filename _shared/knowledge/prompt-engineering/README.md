# Knowledge: prompt-engineering

## 1. 何を保存する場所か
プロンプト設計の技法・パターン(良いプロンプトの書き方に関する知識)。実際に使うプロンプトそのものは含めない。

## 2. 保存してはいけない情報
実際のプロンプト本文(→`_shared/prompts/`)、ブランド固有の指示文言。

## 3. 誰が参照するか
COO、プロンプトを設計する全エージェント。

## 4. 更新ルール
新しいプロンプト技法を試して効果が確認できたら追加する。

## 5. 推奨する情報源(Sources)
`official`, `documentation`, `experiment` ― 具体例: 各AIベンダーの公式プロンプトガイド、実務での検証結果

## 6. 関連Knowledge
- `ai/`
- `chatgpt/`
- `gemini/`
- `perplexity/`

## 7. 機密情報の扱い
技法・パターンは`public`/`internal`。社内限定の工夫は`internal`にする。

## 8. 重複防止ルール
既存の技法カタログに追記する形を優先し、似た技法の記事を乱立させない。
