# Knowledge: design

## 1. 何を保存する場所か
デザイン理論・配色・レイアウトの原則(一般的なデザインの型)。

## 2. 保存してはいけない情報
特定ブランドのカラー・フォント・実際のデザインルール(→`_company/brands/{ブランドID}/brand-brief.md`)。

## 3. 誰が参照するか
`ig-content-planning`、将来のデザイン関連エージェント。

## 4. 更新ルール
実務で効果が高いと確認できたデザインパターンを追加・更新する。

## 5. 推奨する情報源(Sources)
`book`, `industry-report` ― 具体例: デザイン理論の専門書、Canva/Pinterest等のトレンド分析

## 6. 関連Knowledge
- `branding/`
- `copywriting/`
- `pinterest/`
- `canva/`

## 7. 機密情報の扱い
一般理論のため基本`public`/`internal`。未公開の自社デザイン資産の話は含めない。

## 8. 重複防止ルール
既存の配色・レイアウト理論と重複する場合は新規作成せず追記する。
