# Knowledge: _templates

## 1. 何を保存する場所か
Knowledge記事そのものの書き方・型に関する知識(良いKnowledge記事とは何か、フロントマターの埋め方のコツ等)。実際のテンプレートファイルは同階層の[`knowledge-template.md`](../knowledge-template.md)を参照。

## 2. 保存してはいけない情報
特定カテゴリの実際のKnowledge内容そのもの。

## 3. 誰が参照するか
Knowledgeを新規作成する全エージェント・COO。

## 4. 更新ルール
Knowledge運用ルールが変わったら(例: フロントマター項目の追加)更新する。

## 5. 推奨する情報源(Sources)
`internal`, `experiment`

## 6. 関連Knowledge
(全カテゴリ共通の運用知識のため、特定カテゴリとの強い関連はなし)

## 7. 機密情報の扱い
運用ノウハウのため基本`internal`。

## 8. 重複防止ルール
[`README.md`](../README.md)(Knowledge Layer全体の運用ルール)と内容が重複しないよう、ここは「個々の記事の書き方」という執筆技術に限定する。
