# Knowledge: marketing

## 1. 何を保存する場所か
マーケティング全般の理論・フレームワーク(AIDA、STP、ファネル設計等)。特定チャネル固有の知識はそれぞれのフォルダ(`instagram/`, `youtube/`等)へ置く。

## 2. 保存してはいけない情報
特定事業の実際の売上・広告費等の実績数値(→事業側の`data/`・`reports/`)。

## 3. 誰が参照するか
Instagram固有戦略機能(planned / unassigned。および将来、全事業の戦略系エージェント)。

## 4. 更新ルール
理論そのものの陳腐化は少ないが、実務で有効性が確認できたら`confidence`を`verified`へ更新する。

## 5. 推奨する情報源(Sources)
`book`, `research-paper`, `industry-report` ― 具体例: マーケティングの定番書籍、業界レポート

## 6. 関連Knowledge
- `psychology/`
- `copywriting/`
- `branding/`
- `analytics/`

## 7. 機密情報の扱い
一般理論は`public`/`internal`。自社の戦略実績と紐づく内容は含めない。

## 8. 重複防止ルール
同じフレームワークの重複記事を作らない。既存記事へ具体例を追記する形を優先する。
